  \begin{block}{Abstract}

    This poster introduces a \textbf{Model Compression API} for fast and precise inference in ExecuTorch --- an alternative to \texttt{torch.ao} that improves the quality of quantized and compressed model predictions using a rich collection of data-aware and data-free algorithms from the \textbf{Neural Network Compression Framework (NNCF)}.

    \heading{Overview}

    NNCF is an open-source framework that provides a suite of advanced compression algorithms for neural networks, enabling efficient deployment on resource-constrained devices. Within the ExecuTorch ecosystem, NNCF exposes two primary APIs:

    \begin{itemize}
      \item \textbf{\texttt{quantize\_pt2e}} --- Post-Training Quantization API. Smooth Quant and Bias Correction algorithms can be applied alongside any \texttt{torch.ao} Quantizer to improve the output quality of image and speech generation models.
      \item \textbf{\texttt{compress\_pt2e}} --- Weight Compression API. AWQ, GPTQ, and scale estimation algorithms help reduce the perplexity of compressed LLMs with little to no impact on model performance. Paired with the OpenVINO quantizer, we were able to significantly improve the quality of compressed Llama~3.2 model generations for the OpenVINO ExecuTorch backend.
    \end{itemize}

    Both APIs integrate seamlessly within ExecuTorch samples and the Optimum ExecuTorch repository, providing a drop-in enhancement for existing quantization workflows.

  \end{block}

  \begin{block}{Methodology: torch.ao quantization pipeline}

    PyTorch~2 Export Quantization provides a modular, graph-based post-training quantization (PTQ) workflow built on top of \texttt{torch.export} and \texttt{torch.fx}. The pipeline decouples three concerns --- quantizer configuration, graph annotation, and model conversion --- making it straightforward to plug in different backends.

    \begin{figure}
      \centering
      \includesvg[width=0.95\colwidth]{figures/quantization_flow}
      \caption{torch.ao Post-Training Quantization flow.}
    \end{figure}

    \heading{Pipeline Steps}

    \begin{enumerate}
      \item \textbf{Export} --- \texttt{torch.export.export\_for\_training()} captures the model as a full-graph \texttt{torch.fx.GraphModule}, preserving autograd semantics needed for calibration.
      \item \textbf{Prepare} --- \texttt{prepare\_pt2e(model, quantizer)} inserts observer / fake-quantize nodes into the graph according to the backend-specific \texttt{Quantizer}.
      \item \textbf{Calibrate} --- Run representative data through the prepared model so observers collect activation statistics (min/max, histograms, etc.).
      \item \textbf{Convert} --- \texttt{convert\_pt2e(model)} replaces observers with actual quantize/dequantize operations, producing a quantized \texttt{GraphModule} ready for lowering.
    \end{enumerate}

    A \texttt{Quantizer} encapsulates all backend-specific decisions --- which operators to quantize, which dtypes and schemes to use, and how to fuse patterns. PyTorch ships several built-in quantizers (e.g.\ \texttt{XNNPACKQuantizer}, \texttt{X86InductorQuantizer}); NNCF contributes the \texttt{OpenVINOQuantizer} for the OpenVINO backend.

  \end{block}


\end{column}

\begin{column}{\colwidth}


  \begin{block}{Compression API: \texttt{compress\_pt2e} \& \texttt{quantize\_pt2e} }

    NNCF provides two complementary APIs within the \texttt{nncf.experimental.torch.fx} module that extend the standard \texttt{torch.ao} quantization with advanced data-aware algorithms.


    \heading{\texttt{compress\_pt2e} --- Weight Compression}

    Applies weight compression to a \texttt{torch.fx.GraphModule}, targeting LLM deployment. Key algorithm flags:

    \begin{itemize}
      \item \textbf{AWQ} (\texttt{awq=True}) --- Activation-aware Weight Quantization that finds optimal per-channel scales to minimize quantization error based on activation distributions.
      \item \textbf{Scale Estimation} (\texttt{scale\_estimation=True}) --- Estimates optimal quantization scales for 4-bit layers to further reduce perplexity of compressed LLMs.
      \item \textbf{GPTQ} (\texttt{gptq=True}) --- Applies the GPTQ algorithm for accurate layer-wise weight quantization using second-order information.
      \item \textbf{LoRA Correction} (\texttt{lora\_correction=True}) --- Applies Low-Rank Adaptation corrections to compensate for weight compression errors.
    \end{itemize}

    Currently, \texttt{compress\_pt2e} supports the \texttt{OpenVINOQuantizer} and the OpenVINO backend. Support for third-party quantizers can be added by implementing a lightweight NNCF quantizer adapter.

    \heading{\texttt{quantize\_pt2e} --- Post-Training Quantization}

    Can be applied on top of \emph{any} \texttt{torch.ao} Quantizer to improve the accuracy of the already-quantized model. The API wraps the standard \texttt{prepare\_pt2e}/\texttt{convert\_pt2e} flow and augments it with data-aware algorithms:

    \begin{enumerate}
      \item \textbf{SmoothQuant} --- Reduces activation quantization error by inserting smoothing scales before weighted layers. Controlled via \texttt{smooth\_quant=True} and per-layer alpha parameters for convolution and matmul operations.
      \item \textbf{BiasCorrection} --- Corrects bias shifts introduced by quantization. \texttt{fast\_bias\_correction=True} (default) uses a faster method; set to \texttt{False} for higher accuracy, or \texttt{None} to disable.
    \end{enumerate}

    This means existing quantization pipelines can benefit from NNCF's accuracy-boosting algorithms without changing the quantizer --- simply replace the standard \texttt{torch.ao} quantization call with \texttt{quantize\_pt2e} and supply a calibration dataset.


  \end{block}

  \begin{block}{OpenVINOQuantizer}

    The \texttt{OpenVINOQuantizer} from NNCF is a \texttt{torch.ao} Quantizer tailored for the OpenVINO backend. It unlocks the full potential of low-precision OpenVINO kernels by placing quantizers designed specifically for OpenVINO operator implementations.

    \heading{Configuration}

    \texttt{OpenVINOQuantizer} exposes several parameters for fine-grained control:

    \begin{itemize}
      \item \textbf{\texttt{preset}} --- \texttt{PERFORMANCE} (default): symmetric quantization of weights and activations. \texttt{MIXED}: symmetric weights with asymmetric activations, recommended for non-ReLU models (ELU, PReLU, GELU).
      \item \textbf{\texttt{model\_type}} --- Set to \texttt{Transformer} for Transformer-based architectures (BERT, Llama, etc.) to preserve accuracy with a specialized quantization scheme.
      \item \textbf{\texttt{target\_device}} --- Optimize for a specific hardware target: \texttt{ANY} (default), \texttt{CPU}, \texttt{CPU\_SPR}, \texttt{GPU}, or \texttt{NPU}.
    \end{itemize}

  \end{block}
