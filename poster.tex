        We present a \textbf{Model Compression API} for ExecuTorch, powered by the \textbf{Neural Network Compression Framework (NNCF)}.
        It extends the standard \textbf{torch.ao} quantization pipeline with data-aware algorithms --- \textbf{quantize\_pt2e} for post-training quantization and \textbf{compress\_pt2e} for weight compression.
        \par

        We demonstrate that the model compression with the \textbf{NNCF API} yields significant speedups on Llama~3.2, YOLO26, and Stable Diffusion while preserving target metrics --- for the \textbf{OpenVINO ExecuTorch backend}.
        \par

        The NNCF compression API operates on the quantization annotations provided by a given \textbf{torch.ao} Quantizer --- rather than on backend-specific internals --- so any vendor backend can leverage its data-aware algorithms without altering the compressed model architecture.

      \end{block}

      \begin{block}{NNCF Compression API}

        NNCF's \textbf{quantize\_pt2e} and \textbf{compress\_pt2e} extend the standard torch.ao quantization pipeline with data-aware algorithms while preserving compatibility with the \textbf{torch.ao} Quantizer.

        \begin{figure}
          \centering
          \includesvg[width=0.65\colwidth]{figures/quantization_flow}
          \caption{NNCF Compression API scheme: \textbf{quantize\_pt2e} / \textbf{compress\_pt2e}.}
        \end{figure}

        Unlike the standard \textbf{torch.ao} flow, NNCF embeds a \textbf{calibration loop} inside the quantization pipeline, enabling data-aware algorithms to be applied \textbf{sequentially} --- each algorithm refines the model using calibration data before the next one runs.
        A \textbf{data-free} mode is also supported: when no calibration dataset is provided, \textbf{compress\_pt2e} applies weight-only compression with Mixed Precision and AWQ algorithms.

      \end{block}


    \end{column}

    \begin{column}{\colwidth}

      \begin{block}{Weight Compression --- \textbf{compress\_pt2e}}

        \heading{}

        Applies weight compression to a \textbf{torch.fx.GraphModule}, targeting LLM deployment. Key algorithms:

        \begin{itemize}
          \item \textbf{Data-Free Compression}  --- Performs weight compression without any calibration data, relying solely on the pretrained weights. Can be used together with the \textbf{AWQ} and \textbf{Mixed Precision} algorithms.
          \item \textbf{Mixed Precision}  --- Assigns different bit-widths (e.g.\ INT4/INT8) to individual layers based on their sensitivity.
                The data-free \textbf{Weight Quantization Error} and four activation-aware metrics (\textbf{Hessian}, \textbf{Mean/Max Variance}, and \textbf{Mean Magnitude}) are available.
          \item \textbf{Scale Estimation}  --- Estimates scale to minimize the layer-wise output error for INT4 weight layers, iteratively refining the scales on a calibration subset.
                \begin{figure}
                  \centering
                  \includesvg[width=.9\colwidth]{figures/scale_estimation}
                  \caption{Scale Estimation finds per-group scale factors that minimize the error of compressed layer outputs by grid searching a list of scale mutations normalized by the layer inputs.}
                \end{figure}
          \item \textbf{AWQ}  --- Activation-aware Weight Quantization that finds per-channel scales to minimize quantization error based on activation distributions~\cite{awq}.
                %      \begin{figure}
                %        \centering
                %        \includegraphics[width=0.95\colwidth]{figures/AWQ_scheme.png}
                %        \caption{Magnitude of the input activations and weights of a linear layer in OPT-13B before and after SmoothQuant.}
                %      \end{figure}
        \end{itemize}

        %Currently, \textbf{compress\_pt2e} supports the \textbf{OpenVINOQuantizer} and the OpenVINO backend. Support for third-party quantizers can be added by implementing a lightweight NNCF quantizer adapter.

      \end{block}
      \begin{block}{Post-Training Quantization --- \textbf{quantize\_pt2e}}

        Can be applied on top of \emph{any} \textbf{torch.ao} Quantizer to improve the accuracy of the quantized models. Key algorithms:

        \begin{enumerate}
          \item \textbf{SmoothQuant} --- Reduces activation quantization error by inserting smoothing scales before weighted layers~\cite{smoothquant}.

                \begin{figure}
                  \centering
                  \includegraphics[width=0.8\colwidth]{svg-inkscape/sq_scheme.png}
                  \caption{Magnitude of the input activations and weights of a linear layer in OPT-13B before and after SmoothQuant. Figure source\cite{smoothquant}.}
                \end{figure}
          \item \textbf{BiasCorrection} --- Compares quantized and original layer outputs layer-by-layer and adjusts convolution biases to align them, compensating for the error introduced by quantization~\cite{bias_correction}.
        \end{enumerate}


        % This means existing quantization pipelines can benefit from NNCF's accuracy-boosting algorithms without changing the quantizer --- simply replace the standard \textbf{torch.ao} quantization call with \textbf{quantize\_pt2e} and supply a calibration dataset.

      \end{block}
